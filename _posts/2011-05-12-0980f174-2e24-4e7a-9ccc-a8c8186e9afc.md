今天的 Google I/O 大会上，Chrome 开发团队组织了一场回复观众提问的小交流会。

大多数人都感兴趣的话题自然是：chrome 接下来还会有什么变化？

<a rel="attachment wp-att-11769" href="http://www.chromi.org/archives/11765/aac">![](http://img.chromi.org/2011/05/aac.jpg "google i/o Chrome")</a>

说到变化，那 Chrome 团队自己认为最让人期待的新内容会是哪些呢？
除了一些大方向的新变化（如 HTML 和 CSS 的进步），当然也有非常具体的内容，虽然都早有耳闻：

**1.个人化服务
**Chrome将会有更进一步的个人化服务，可以把更多的个人浏览信息推向云端同步。
其中个人化服务里最重要的一项，当属多用户配置功能，这个功能在内部就被简单地叫做 Profile ，可以允许一个 Chrome 里登陆多个账号信息，分别有各自的扩展，皮肤，记录等。其实，这部分内容早已尝试[登陆 Chromium](http://www.chromi.org/archives/11519) 了

**2.标签同步
**这个功能宣传了好久了，可以方便不同的设备间同步或推送标签页（标签页都云了以后，可以说是“无缝切换”了吧）但 Chrome 团队还在纠结着究竟以什么形式来实现最好。

**3.Native Client 和 扩展内部数据同步
**组员说 [Native Client](http://www.chromi.org/archives/10512) 还是要等今年晚点的时候才能正式出现，这个非常关键，是沟通本地应用和 Web 端的重要一环。此外也在开始着手处理扩展内部数据的同步工作

**4.ARM 版的 Chrome OS
**据说事实上已经差不多可以运行了，但是 Google 当前主要还是要把精力施注到合作伙伴三星和 Acer 上（当然都是 Intel inside ）。说到低功耗又轻量的机子，还是要看 ARM 。

**5.Canary for Linux ?
**这个真没有，考虑到 Linux 用户都自己调着 Chromium 玩⋯当然，将来是有可能会出现。[Mac 版](http://www.chromi.org/archives/11649)的出来也刚好几天。
