著名科技博客 TechCrunch 又[带来了猛料](http://techcrunch.com/2010/08/14/chrome-os-verizon/)，今天发表文章说总总迹象显示美国的移动网络运营商 Verizon 正在与 Google 合作搞一个采用 Chrome OS 操作系统的设备。

正如下面这张 Chromium OS 在 [Google Code](http://code.google.com/p/chromium-os/issues/list?can=2&q=verizon&colspec=ID+Stars+Pri+Area+Type+Status+Summary+Modified+Owner+Mstone&x=mstone&y=area&cells=tiles) 页面上的截图中你所看到的，很多地方都表明 Chrome OS 不仅仅会有3G网络连接，而且 Verizon 会成为 Chrome OS 设备的合作伙伴。

<a href="http://img.chromi.org/2010/08/vzw-google-chrome-ui-list-4801.png">![](http://img.chromi.org/2010/08/vzw-google-chrome-ui-list-4801.png "vzw-google-chrome-ui-list-4801")</a>

[IntoMobile](http://www.intomobile.com/2010/08/15/google-adding-verizon-specific-references-to-chrome-os-code-mobile-device-in-the-works/) 在他们的报道中就说的更明白了：



>Google Code 页面上的内容清楚的表明了 Google 曾经在让 Chrome OS 支持无线连接方面而努力，而最近该项目的焦点被转到了3G连接的UI组件以及3G移动宽带账号管理方面。虽然从底层的代码来看，其实现方式是通过 CDMA 甚至是 WiMAX，但最近的项目更新则直接指向了 Verizon 无线。



确实这个事情也算不上什么大新闻，因为最近几年 Verizon 和 Google 的关系一直比较暧昧。作为一个主要的移动运营商，Verizon 已经在带头搞基于 Google Android 操作系统的平板电脑，Google 将这个大主儿当成一个重要的 Chrome OS 合作伙伴也是理所当然的选择。

那么我们不难想象，未来 Google 是不是也有可能像 iPad/iPhone 那样，在美国与一个指定的运营商合作，然后将这种模式引入到其他国家呢，因为对于用户来说反正用这种设备就是要上网的，能省点儿钱就省点儿呗。

我们再进一步YY，既然底层代码显示采用 CDMA 的网络，那么未来中国电信是不是也有戏呢，能依靠 Chrome OS 平板设备和联通小争一把也不是什么坏事儿。
